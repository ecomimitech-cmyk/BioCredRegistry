Phase A — Traditional Flow First

Fastest path to a public, working product. Agencies review documents the traditional way. No heatmap required.

Priority
1
Deploy Supabase project (live)
Create the production project, run schema migrations, apply RLS policies, configure auth.
2
Biologist upload flow
Account creation → draft profile → upload credential and evidence documents → submit for review. Controlled, step-by-step onboarding.
3
Reviewer / agency interface
Reviewers access submitted documents, approve or reject with notes, full audit trail. No heatmap required to complete a review.
4
Public profile page (live)
Displays verified credentials + profile summary, closely matching your draft design. Clean, neutral, no scoring language.

Phase B — Heatmap as Optional Layer

Added after Phase A is live. Agencies can opt out. Biologists opt in. No paywalls, no premium tiers needed.

After Phase A
6
Connect translation engine to live data
Run the pipeline on verified uploaded documents to generate the render payload. Server-side only — browser never computes the math.
7
Heatmap view toggle on profile page
Optional display layer — off by default, toggled on by choice. Agencies can stay in traditional review mode at any time.
8
Final visual polish on pentagon chart
Cosmetic refinements aligned with the established visual grammar. Confirmed readable and consistent.
9
"What-if" query prototype (future feature)
Exploratory — e.g. "what if I completed a PhD in X?" Simulated profile shift. Informational only, no scoring implied.
One database throughout. Both flows — traditional document review and optional heatmap — run inside the same single Supabase project with clear table separation and role-based access. No duplication, no extra cost. Everything that has been built carries forward into production without rework.
