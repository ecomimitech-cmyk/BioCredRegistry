# BioCred Registry — Category Definition Reference

## Purpose

Defines what qualifies as evidence for each of the five credential domains.
Prevents inconsistent interpretation and ensures neutral, structured data entry.

**Governing Principle**: These domains measure documented scope, not skill quality.

---

## Domain 1: Field Experience

**Definition**: Documented applied work conducted in real-world settings.

| Counts As | Does Not Count As |
|-----------|-------------------|
| Biological monitoring | Classroom-only training |
| Construction monitoring | Desk-based analysis without field execution |
| Field surveys and inventories | Lab work without field component |
| Mitigation and compliance monitoring | Observation from vehicle only |
| On-site data collection and reporting | Reviewing others' field data |
| Seasonal or long-term field assignments | Conference attendance |
| Habitat assessments | Reading field guides |
| Wetland delineation | Remote sensing without ground-truthing |
| Vegetation mapping | |

**Purpose**: Captures hands-on operational experience without implying proficiency.

---

## Domain 2: Regulatory Experience

**Definition**: Exposure to regulatory frameworks and compliance contexts.

| Counts As | Does Not Count As |
|-----------|-------------------|
| CEQA / NEPA project involvement | Authority to issue permits |
| CESA / ESA context exposure | Legal interpretation or enforcement power |
| Lake and Streambed Alteration Agreement work | General administrative tasks |
| Incidental Take Permit support | Policy lobbying or advocacy |
| Permit condition monitoring or reporting | Reading regulations without application |
| Agency coordination or documentation support | General consulting unrelated to compliance |
| Biological assessment authorship | |
| Environmental impact report contribution | |
| Section 404 / Clean Water Act work | |

**Purpose**: Describes familiarity with regulatory environments without implying authority.

## Domain 3: Academic / Technical

**Definition**: Formal education and technical skill development.

| Counts As | Does Not Count As |
|-----------|-------------------|
| Degrees and formal coursework | Informal training unless documented |
| Research projects | Conference attendance without presentation |
| Publications or technical reports | Self-study without credential |
| Laboratory experience | Webinars without certification |
| GIS, statistics, modeling, or analytical tools | Tool usage without formal training |
| Methods development or validation | Reading publications |
| Thesis or dissertation research | |
| Peer review participation | |
| Teaching or curriculum development | |

**Purpose**: Represents formal knowledge foundations, not applied judgment.

## Domain 4: Species Depth

**Definition**: Depth of experience with specific taxa or species groups.

| Counts As | Does Not Count As |
|-----------|-------------------|
| Taxon-specific monitoring | Casual exposure without documentation |
| Listed species exposure (documented) | Zoo or captive-only observation |
| Species handling (documented and authorized) | Pet ownership |
| Protocol-specific training | Identifying species from photos only |
| Repeated work with same taxa over time | One-time sighting without documentation |
| Species-specific survey techniques | General ecology coursework |
| Handling permits held | |
| Breeding/nesting monitoring | |
| Translocation project participation | |
| Population assessment work | |

**Purpose**: Communicates specialization scope without ranking expertise.

## Domain 5: Leadership / Responsibility Scope

**Definition**: Scope of responsibility held during biological work.

| Counts As | Does Not Count As |
|-----------|-------------------|
| Lead biologist roles | Informal leadership without documentation |
| Crew or team oversight | Participation without supervisory role |
| QA/QC responsibility | Attending meetings |
| Reporting authority | Advisory roles without decision authority |
| Client or agency-facing coordination | Volunteering without formal role |
| Training or mentoring roles | Self-directed work without team context |
| Principal Investigator designation | |
| Project management for biological scope | |
| Multi-team coordination | |

**Purpose**: Reflects responsibility level, not superiority.

---

## Cross-Domain Rules

1. A single activity **can** generate evidence in multiple domains
   - Example: Leading a CEQA field survey → Field + Regulatory + Leadership
2. Each evidence record is filed under **one** domain only
   - Multi-domain activities create separate records per domain
3. No domain has priority over another
4. No composite score is calculated from domain data
5. Domains are descriptive, not evaluative

---

## Verification Note

Category placement is based on **documentation provided**, not self-assessment.
If a biologist claims field experience, the credential record must reference
supporting documentation (project reports, employer confirmation, etc.).
